# 🚀 دليل تشغيل مشروع Hatim ERP

## الخطوات المطلوبة

### 1️⃣ إنشاء قاعدة البيانات

افتح MySQL وقم بتنفيذ الأمر التالي:

```sql
CREATE DATABASE hatim_erp CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

أو استخدم الملف الجاهز:
```bash
mysql -u root -p12255 < create-database.sql
```

### 2️⃣ تشغيل المشروع لإنشاء الجداول

```bash
cd backend
pnpm start:dev
```

انتظر حتى يظهر:
```
🚀 Application is running on: http://localhost:3000/api
```

ثم أوقف المشروع بالضغط على `Ctrl+C`

### 3️⃣ تعبئة البيانات الوهمية

```bash
pnpm seed
```

سيتم إنشاء:
- ✅ 500 منتج متنوع
- ✅ 10 موردين
- ✅ 100 عميل
- ✅ 4 مستخدمين
- ✅ 3 صناديق نقدية
- ✅ 50 فاتورة شراء
- ✅ 200 فاتورة بيع
- ✅ 30 حالة تلف
- ✅ 20 تسوية جرد
- ✅ 50 مصروف
- ✅ 100 حركة دفع

### 4️⃣ تشغيل المشروع

```bash
pnpm start:dev
```

## 📡 API Endpoints

### المنتجات
- `GET /api/products` - قائمة المنتجات
- `POST /api/products` - إضافة منتج
- `GET /api/products/:id` - تفاصيل منتج
- `PUT /api/products/:id` - تعديل منتج
- `DELETE /api/products/:id` - حذف منتج

### المخزون
- `GET /api/inventory/stock` - المخزون الكامل
- `GET /api/inventory/stock/:productId` - مخزون منتج محدد
- `GET /api/inventory/history/:productId` - تاريخ حركة منتج

### المشتريات
- `POST /api/purchases` - إنشاء فاتورة شراء
- `GET /api/purchases` - قائمة المشتريات
- `GET /api/purchases/:id` - تفاصيل فاتورة
- `PUT /api/purchases/:id/receive` - استلام البضاعة
- `PUT /api/purchases/:id/post` - ترحيل للمخزون

### المبيعات
- `POST /api/sales` - إنشاء فاتورة بيع
- `GET /api/sales` - قائمة المبيعات
- `GET /api/sales/:id` - تفاصيل فاتورة
- `PUT /api/sales/:id/close` - إغلاق فاتورة

### العملاء والموردين
- `GET /api/customers` - قائمة العملاء
- `POST /api/customers` - إضافة عميل
- `GET /api/suppliers` - قائمة الموردين
- `POST /api/suppliers` - إضافة مورد

### المصروفات
- `GET /api/expenses` - قائمة المصروفات
- `POST /api/expenses` - تسجيل مصروف

### المحاسبة
- `GET /api/cash-accounts` - الصناديق النقدية
- `POST /api/cash-accounts` - إضافة صندوق
- `GET /api/payment-transactions` - حركات الدفع
- `POST /api/payment-transactions` - تسجيل حركة

### المستخدمين
- `POST /api/auth/login` - تسجيل الدخول
- `POST /api/auth/register` - تسجيل مستخدم جديد

## 📝 ملاحظات مهمة

1. **المخزون Event-Based**: لا يوجد عمود stock في جدول المنتجات، كل شيء يُحسب من الـ Events
2. **UUIDv7**: جميع الـ IDs تستخدم UUIDv7 ومخزنة كـ BINARY(16)
3. **العلاقات**: جميع البيانات مترابطة ومنطقية
4. **التواريخ**: البيانات موزعة على آخر 90 يوم

## 🛠️ استكشاف الأخطاء

### خطأ في الاتصال بقاعدة البيانات
تأكد من:
- MySQL يعمل
- بيانات الاتصال صحيحة في `.env`
- قاعدة البيانات `hatim_erp` موجودة

### خطأ أثناء الـ seed
- تأكد من تشغيل المشروع مرة واحدة أولاً لإنشاء الجداول
- تأكد من عدم وجود بيانات قديمة (احذف قاعدة البيانات وأعد إنشاءها)
